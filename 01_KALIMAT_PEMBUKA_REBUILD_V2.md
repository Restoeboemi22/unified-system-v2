# KALIMAT PEMBUKA REBUILD V2

Dokumen ini adalah pegangan pembuka saat mulai bekerja di workspace `D:\Unified-System-V2`.

## Tujuan

Gunakan teks di bawah ini saat membuka workspace baru atau saat perlu mengulang konteks kerja:

```text
Saya sudah membuka workspace baru di `D:\Unified-System-V2`.

Sebelum melakukan apa pun, baca file `E:\Aplikasi Android\Portal Sekolah\Siap Tempur\GAS\Kitab Suci\00_PERINTAH_MULAI_REBUILD_V2.md` dan jadikan isinya sebagai instruksi kerja utama.

Setelah itu, baca juga file lokal `01_KALIMAT_PEMBUKA_REBUILD_V2.md` dan `02_PROGRES_HARI_INI_DAN_LANGKAH_BERIKUTNYA.md` di root workspace ini, lalu jadikan keduanya sebagai pegangan konteks kerja terkini.

Pahami bahwa:
- workspace `D:\Unified-System-V2` adalah tempat membangun repo baru V2
- `E:\Aplikasi Android\Portal Sekolah` adalah sumber referensi sistem aktif lama
- struktur project lama tidak boleh dicopy mentah ke workspace baru
- blueprint V2 adalah acuan utama
- pengembangan harus bertahap, tidak melompat sembarangan
- fondasi utama backend V2 sudah terbentuk
- POC minimum admin berbasis web sudah ada dan sedang disempurnakan
- fokus saat ini adalah menutup gap blueprint yang tersisa, menstabilkan verifikasi lokal, dan melanjutkan tahap berikutnya secara terstruktur

Setelah membaca file-file itu, lanjutkan pengembangan Unified-System V2 di workspace ini sesuai blueprint, progres terakhir, dan urutan tahapan yang paling aman.
```

## Cara Pakai

Gunakan dokumen ini untuk memastikan konteks awal selalu sama:
- baca instruksi utama dari referensi blueprint
- baca progres lokal terbaru di root workspace
- baca ringkasan aturan kerja singkat di `03_ATURAN_KERJA_SINGKAT.md` jika perlu versi 1 halaman
- lanjutkan pekerjaan secara bertahap
- jangan melompat ke perubahan besar tanpa dasar dari blueprint dan progres terakhir

Jadikan semuanya sebagai pegangan kerja, lalu lanjutkan Unified-System V2 secara bertahap sesuai blueprint dan progres terakhir.
